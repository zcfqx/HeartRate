# HeartRate Monitor 前端样式规范

> 本文档为 WPF/XAML 桌面应用的 UI 样式规范，定义了颜色、字体、间距、控件等设计标准。
> 所有界面修改应遵循本规范，确保视觉一致性。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **深色优先** | 应用以深色半透明悬浮窗为核心形态，所有颜色定义基于深色主题 |
| **信息密度适中** | 主窗口仅 300×420 像素，需在有限空间内清晰展示心率、统计、图表 |
| **状态驱动视觉** | 心率数值、连接状态、心率区间等核心数据通过颜色变化传达状态 |
| **极简可切换** | 支持极简模式，隐藏非核心信息，仅保留 BPM 数值 |

## 2. 颜色体系

### 2.1 主题色板（Theme.xaml）

所有颜色通过 `StaticResource` 引用，修改时只需改 Theme.xaml 中的定义。

| 资源键 | 色值 | 用途 |
|--------|------|------|
| `PrimaryColor` | `#6366F1` | 主色调（Indigo 紫蓝），用于按钮、高亮、聚焦边框 |
| `PrimaryLightColor` | `#818CF8` | 主色调亮色变体，用于 hover 状态、分区标题 |
| `BackgroundColor` | `#1A1A2E` | 窗口主背景色（深蓝黑） |
| `SurfaceColor` | `#16213E` | 表面色/次级背景（深蓝） |
| `CardColor` | `#0DFFFFFF` | 卡片背景（8% 白色透明叠加） |
| `TextPrimaryColor` | `#FFFFFF` | 主文本色（纯白） |
| `TextSecondaryColor` | `#B3FFFFFF` | 次要文本色（70% 白色透明） |
| `TextMutedColor` | `#66FFFFFF` | 弱化文本色（40% 白色透明） |
| `BorderColor` | `#33FFFFFF` | 通用边框色（20% 白色透明） |

### 2.2 覆盖层色

| 色值 | 用途 |
|------|------|
| `#141422` | 窗口覆盖层基色，配合 `OverlayOpacity`（0.3–1.0）控制窗口透明度 |

### 2.3 语义色

#### 心率区间色

| 区间 | 心率范围 (BPM) | 文本色 | 区间标签色 |
|------|----------------|--------|-----------|
| 静息 (Rest) | < 100 | `#FFFFFF` 白色 | `#6366F1` 紫色 |
| 燃脂 (FatBurn) | 100 – 139 | `#22C55E` 绿色 | `#22C55E` 绿色 |
| 有氧 (Cardio) | 140 – 169 | `#F59E0B` 琥珀色 | `#F59E0B` 琥珀色 |
| 极限 (Peak) | 170 – 199 | `#F97316` 橙色 | `#F97316` 橙色 |
| 最大 (Maximum) | ≥ 200 | `#EF4444` 红色 | `#EF4444` 红色 |

#### 连接状态色

| 状态 | 色值 | 含义 |
|------|------|------|
| 已连接 | `#22C55E` 绿色 | 正常工作 |
| 扫描中/连接中/重连中 | `#F59E0B` 琥珀色 | 过渡状态 |
| 未连接/其他 | `#EF4444` 红色 | 异常/断开 |

### 2.4 图表色（LiveCharts2 / SkiaSharp）

| 元素 | 色值 | 说明 |
|------|------|------|
| 心率折线 | `CornflowerBlue` (`#6495ED`) | 线宽 2 |
| RR 间期折线 | `MediumPurple` (`#9370DB`) | 线宽 1.5 |
| 坐标轴标签 | `WhiteSmoke` | 通用图表文字色 |
| 坐标轴分隔线 | `WhiteSmoke` | 网格线色 |

### 2.5 颜色使用规则

```
规则 1: 所有主题色必须通过 Theme.xaml 的 StaticResource 引用，禁止硬编码色值
规则 2: 语义色（心率区间、连接状态）由 Converter 动态计算，修改时改 Converters.cs
规则 3: 图表色在 ViewModel 中设置，修改时改 MainViewModel.cs
规则 4: 新增颜色必须先在 Theme.xaml 中定义 Color + SolidColorBrush，再引用
```

## 3. 字体规范

### 3.1 字体族

| 场景 | 字体 | 备选 |
|------|------|------|
| BPM 数值显示 | `Segoe UI Light` | — |
| 其他所有文字 | 系统默认（`Segoe UI`） | — |

### 3.2 字号层级

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Display | 72 | Bold | BPM 数值（主窗口核心数据） |
| H1 | 22 | SemiBold | 统计数值（最低/平均/最高） |
| H2 | 18 | SemiBold | RR间期数值、信号强度数值、心率区间标签 |
| H3 | 16 | Normal | BPM 单位标签 |
| Body | 15 | SemiBold | 设置窗口标题 |
| Body | 14 | Normal/SemiBold | 按钮文字（主要操作） |
| Body | 13 | Normal | 设置项标签、按钮文字（次要操作） |
| Caption | 12 | SemiBold | 设置分区标题 |
| Small | 11 | Normal | 设置项描述文字 |
| Tiny | 10 | Normal | 设备名称、最后更新时间 |
| Micro | 9 | Normal | 统计标签（最低/平均/最高等） |

### 3.3 字体使用规则

```
规则 1: BPM 数值是视觉核心，字号 72 + Bold + Segoe UI Light，不可缩小
规则 2: 标签/描述文字字号 ≤ 12pt，使用 Normal 字重
规则 3: 数值/标题文字字号 ≥ 13pt，使用 SemiBold 字重
规则 4: 新增文字元素必须从上述层级中选择，不得随意定义新字号
```

## 4. 间距系统

### 4.1 基础间距单位

采用 **4px 基准网格**，所有间距为 4 的倍数。

| 代号 | 值 | 用途 |
|------|-----|------|
| `xs` | 2px | 紧凑元素间距（列表项 Margin） |
| `sm` | 4px | 窗口外框 Margin、基础间距 |
| `md` | 8px | 标题栏右侧 Margin、按钮内 Padding、圆角小值 |
| `lg` | 12px | 内容区 Margin、图表容器 Margin、卡片底部间距 |
| `xl` | 16px | 卡片内 Padding、按钮水平 Padding、主窗口水平 Margin |
| `2xl` | 20px | 统计区域水平 Margin |

### 4.2 窗口内边距

| 窗口 | 外边距 | 内容区边距 |
|------|--------|-----------|
| MainWindow | 4px | 水平 12px，顶部 4px，底部 10px |
| SettingsWindow | 4px | 水平 16px（卡片内），顶部标题栏 40px |
| DevicePickerWindow | — | 水平 20px，垂直 20px |

### 4.3 间距规则

```
规则 1: 所有间距值必须是 4 的倍数（2px 的 xs 例外）
规则 2: 同一区域内元素间距保持一致
规则 3: 卡片内部统一 Padding="16"
规则 4: 按钮内边距统一 Padding="16,8"
```

## 9. 动画与交互

### 9.1 状态过渡

| 交互 | 效果 |
|------|------|
| 按钮 Hover | 背景色过渡到 PrimaryLightColor |
| 按钮 Pressed | Opacity 降至 0.8 |
| 输入框聚焦 | 边框色过渡到 PrimaryColor |
| 下拉项 Hover | 背景变为 #1AFFFFFF |
| 心率数值变化 | 颜色随数值区间平滑切换 |
| 窗口拖拽 | 标题栏区域可拖拽移动 |

### 9.2 透明度控制

窗口整体不透明度由 `OverlayOpacity` 设置项控制（范围 0.3–1.0），通过 `OverlayOpacityToBrushConverter` 动态计算覆盖层背景色的 Alpha 值。

## 10. 代码组织规范

### 10.1 样式文件结构

```
Styles/
  Theme.xaml              # 全局资源：颜色、画刷、通用按钮样式
Views/
  MainWindow.xaml         # 窗口内样式：TitleBarButton, MainBorder
  SettingsWindow.xaml     # 窗口内样式：SectionCard, ToggleSwitch, InputTextBox 等
  DevicePickerWindow.xaml # 窗口内样式：设备列表项样式
Converters/
  Converters.cs           # 颜色/状态转换逻辑
```

### 10.2 样式定义优先级

```
1. 全局通用样式 → Theme.xaml（颜色、画刷、PrimaryButton）
2. 窗口级样式   → 各窗口 XAML 的 Window.Resources（该窗口专用控件样式）
3. 内联样式     → 仅用于一次性、不可复用的属性设置
```

### 10.3 命名约定

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 颜色键 | `{描述}Color` | `PrimaryColor`, `BackgroundColor` |
| 画刷键 | `{描述}Brush` | `PrimaryBrush`, `TextSecondaryBrush` |
| 样式键 | `{控件类型}{用途}` 或 `{描述}` | `PrimaryButton`, `SectionCard`, `InputTextBox` |

### 10.4 新增样式清单

新增控件时，按以下顺序操作：

```
1. 判断是否需要新颜色    → 是 → Theme.xaml 添加 Color + SolidColorBrush
2. 判断是否可复用全局样式 → 是 → Theme.xaml 添加 Style
3. 判断是否为窗口专用    → 是 → 对应窗口 XAML 的 Window.Resources 添加 Style
4. 仅一次性属性          → 内联设置
```

## 12. 扩展建议

### 12.1 浅色主题适配

当前仅实现了深色主题。若需支持浅色主题（ThemeType.Light），建议：

1. 在 `Styles/` 目录新增 `LightTheme.xaml`
2. 定义相同的资源键，使用浅色色值
3. 在 `App.xaml.cs` 中根据 `ThemeType` 动态切换 ResourceDictionary

### 12.2 样式集中化

当前大量控件样式分散在各窗口 XAML 中（尤其是 SettingsWindow.xaml）。若后续新增窗口需要复用这些样式，建议：

1. 将 `ToggleSwitch`、`InputTextBox`、`InputComboBox`、`SliderStyle`、`ThemedScrollBar` 等提取到 `Styles/Controls.xaml`
2. 在 `App.xaml` 中合并引用

### 12.3 设计令牌常量化

考虑在 `HeartRateMonitor.Core` 或 `HeartRateMonitor.App` 中建立静态类，将关键色值、字号等定义为常量，便于 C# 代码（如 Converters、ViewModel）与 XAML 保持一致：

```csharp
public static class DesignTokens
{
    // 心率区间色
    public const string ColorRest = "#FFFFFF";
    public const string ColorFatBurn = "#22C55E";
    public const string ColorCardio = "#F59E0B";
    public const string ColorPeak = "#F97316";
    public const string ColorMaximum = "#EF4444";

    // 连接状态色
    public const string ColorConnected = "#22C55E";
    public const string ColorTransition = "#F59E0B";
    public const string ColorDisconnected = "#EF4444";
}
```
