# HeartRate Monitor 交互逻辑与前端表现规范

> 本文档定义应用的完整交互流程、状态驱动规则、前端表现逻辑，作为后续 UI 优化和重构的依据。

---

## 1. 全局交互架构

### 1.1 MVVM 数据流

```
用户操作 → Code-Behind 事件 → ViewModel Command → Service → 状态变更
                                                         ↓
                              UI 更新 ← 属性变更通知 ← ObservableProperty
```

**关键约束：**
- 所有业务逻辑必须在 ViewModel 中，Code-Behind 仅处理窗口级操作（拖拽、关闭、打开子窗口）
- Service 通过事件（`EventHandler<T>`）通知 ViewModel，ViewModel 通过 `Dispatcher.Invoke` 回到 UI 线程更新属性
- `[ObservableProperty]` 源生成器自动触发 `PropertyChanged`，驱动 XAML 绑定刷新

### 1.2 窗口层级与生命周期

```
App.OnStartup
  ├── DI 容器构建
  ├── 数据库初始化 (IDataService.InitializeAsync)
  ├── 设置加载 (ISettingsService.LoadAsync)
  ├── MainWindow 创建并显示
  ├── 蓝牙权限请求
  └── 自动重连（如已启用）

MainWindow（Singleton，应用生命周期）
  ├── DevicePickerWindow（Transient，ShowDialog 模态）
  └── SettingsWindow（Transient，ShowDialog 模态）
```

**生命周期规则：**
- MainWindow 是 Singleton，关闭时最小化到托盘而非退出
- 子窗口（DevicePickerWindow、SettingsWindow）是 Transient，每次打开创建新实例
- 子窗口以 `ShowDialog()` 模态方式打开，`Owner` 设为 MainWindow
- 只有 `OnCloseClick`（关闭按钮）设置 `_forceClose = true` 才真正关闭 MainWindow

## 2. MainWindow 交互逻辑

### 2.1 窗口行为

| 行为 | 触发方式 | 实现逻辑 |
|------|----------|----------|
| 拖拽移动 | 标题栏 MouseLeftButtonDown | `DragMove()` + 保存位置 |
| 极简模式拖拽 | 极简模式下整个窗口区域 | `DragMove()` |
| 双击最大化/还原 | 标题栏双击 | 切换 `WindowState` |
| 关闭（最小化） | 关闭按钮 / 系统关闭 | `e.Cancel = true`，最小化到托盘 |
| 真正退出 | 关闭按钮（`_forceClose`） | `Application.Current.Shutdown()` |
| 最小化 | 最小化按钮 | `WindowState.Minimized` |
| 窗口位置持久化 | 拖拽结束、窗口加载 | `ISettingsService.WindowLeft/WindowTop` |

### 2.2 标题栏按钮组

```
[状态圆点] [状态文字]     [极简] [扫描] [设置] [最小化] [关闭]
  │           │              │      │      │       │       │
  │           │              │      │      │       │       └─ _forceClose=true, Close()
  │           │              │      │      │       └─ WindowState.Minimized
  │           │              │      │      └─ 打开 SettingsWindow (ShowDialog)
  │           │              │      └─ 打开 DevicePickerWindow (ShowDialog)
  │           │              └─ ToggleMinimalModeCommand
  │           └─ ConnectionStatus 属性绑定
  └─ ConnectionStateToColorConverter 动态着色
```

### 2.3 极简模式

**触发路径（3条）：**
1. 标题栏极简按钮 → `OnMinimalModeToggle` → `ToggleMinimalModeCommand`
2. BPM 区域点击 → `OnBpmAreaClick` → `ToggleMinimalModeCommand`
3. 设置窗口开关 → `SettingsViewModel.MinimalMode` → `ISettingsService.MinimalMode`

**表现规则：**

| 状态 | 隐藏的元素 | 保留的元素 |
|------|-----------|-----------|
| 正常模式 | — | 全部 9 行内容 |
| 极简模式 | 标题栏、设备名、心率区间、统计、RR/信号、图表、更新时间、底部栏 | 仅 BPM 数值（居中显示） |

**极简模式视觉变化：**
- MainBorder 背景变透明、边框隐藏、圆角归零、Margin 归零
- 标题栏区域变为可拖拽的透明区域（Cursor=SizeAll）
- BPM 数值仍保持颜色随心率变化

### 2.4 心率数据显示流程

```
BLE 设备 → HeartRateReceived 事件
  ↓
OnHeartRateReceived (UI 线程 Dispatcher.Invoke)
  ├── CurrentHeartRate = data.HeartRate
  ├── BpmText = data.HeartRate.ToString()
  ├── RrInterval = data.RRInterval > 0 ? $"{data.RRInterval:F1} ms" : "0 ms"
  ├── LastUpdate = data.Timestamp.ToString("HH:mm:ss")
  ├── UpdateZone(heartRate) → HeartRateZone + ZoneColor
  ├── 图表数据点追加（最多 120 个心率点、60 个 RR 点）
  └── HeartRateService.UpdateHeartRate(data) → 统计更新
```

**BPM 颜色联动规则：**

| 心率范围 | BPM 文本色 | 心率区间文字 | 区间标签色 |
|----------|-----------|-------------|-----------|
| < 100 | `#FFFFFF` 白色 | "静息" | `#6366F1` 紫色 |
| 100–139 | `#22C55E` 绿色 | "燃脂" | `#22C55E` 绿色 |
| 140–169 | `#F59E0B` 琥珀色 | "有氧" | `#F59E0B` 琥珀色 |
| 170–199 | `#F97316` 橙色 | "极限" | `#F97316` 橙色 |
| ≥ 200 | `#EF4444` 红色 | "最大" | `#EF4444` 红色 |

### 2.5 连接状态显示流程

```
BLE Service → ConnectionStateChanged 事件
  ↓
OnConnectionStateChanged (UI 线程)
  ↓
UpdateState(ConnectionState)
  ├── ConnectionStatus = "未连接" / "扫描中..." / "连接中..." / "已连接" / "重连中..."
  ├── ConnectionStatusIcon = 去掉省略号的版本
  ├── IsConnected / IsScanning 布尔值
  ├── ConnectedDevice = 当前设备引用
  ├── DeviceName = 设备名或空
  └── SignalStrength = "xx dBm" 或 "无"
```

**状态圆点颜色映射：**
- 已连接 → `#22C55E` 绿色
- 扫描中/连接中/重连中 → `#F59E0B` 琥珀色
- 未连接/其他 → `#EF4444` 红色

### 2.6 图表行为

| 属性 | 值 |
|------|-----|
| 心率数据窗口 | 最新 120 个点（约 2 分钟，1Hz 采样） |
| RR 数据窗口 | 最新 60 个点 |
| Y 轴范围 | 固定 40–220 BPM |
| X 轴标签 | `HH:mm:ss` 格式时间 |
| 动画速度 | 150ms |
| 线条平滑度 | 心率 0.5，RR 0.3 |
| 图表容器 | 最小高度 80px，圆角 8，背景 `#1AFFFFFF` |

### 2.7 统计数据

| 数据 | 来源 | 更新时机 |
|------|------|----------|
| MinHeartRate | `IHeartRateService.GetStatisticsAsync` | 启动时加载（最近 1 小时） |
| AvgHeartRate | 同上 | 同上 |
| MaxHeartRate | 同上 | 同上 |

## 3. DevicePickerWindow 交互逻辑

### 3.1 窗口生命周期

```
打开:
  MainWindow.OnScanClick
    → App.Services.GetService<DevicePickerWindow>()
    → picker.Owner = this
    → picker.ShowDialog()
    → OnWindowLoaded → ViewModel.AttachEvents()
      ├── 订阅 DeviceDiscovered / ConnectionStateChanged 事件
      ├── 同步当前扫描状态
      └── 加载已发现设备列表

关闭:
  OnWindowClosing → ViewModel.DetachEvents()
    ├── 取消事件订阅
    └── 释放资源
```

### 3.2 设备扫描流程

```
用户点击 "开始扫描"
  → ToggleScanCommand → StartScanAsync
    ├── Devices.Clear() 清空列表
    ├── IsScanning = true
    ├── ScanButtonText = "停止扫描"
    └── BleService.StartScanningAsync()

BLE 发现设备 → DeviceDiscovered 事件
  → OnDeviceDiscovered (UI 线程 BeginInvoke)
    ├── 去重检查 (DeviceId)
    └── Devices.Add(device) → ListBox 自动刷新

用户点击 "停止扫描"
  → ToggleScanCommand → StopScanAsync
    ├── BleService.StopScanningAsync()
    ├── IsScanning = false
    └── ScanButtonText = "开始扫描"
```

### 3.3 设备选择流程

```
用户点击设备列表项
  → OnDeviceClicked (Border.MouseLeftButtonDown)
    → 从 sender 获取 DataContext (BleDevice)
    → ViewModel.SelectDeviceCommand.Execute(device)
      → DeviceSelected 事件触发
        → Code-Behind.OnDeviceSelected
          ├── SelectedDevice = device
          ├── DialogResult = true
          └── Close()

回到 MainWindow.OnScanClick:
  if (result == true && picker.SelectedDevice != null)
    → ViewModel.ConnectAsync(device)
      ├── StopScanAsync()
      ├── BleService.ConnectAsync(device)
      ├── 保存 LastDeviceId
      └── 失败时弹出 MessageBox
```

### 3.4 状态提示区域

| 扫描状态 | StatusDot 颜色 | 提示文字 |
|----------|---------------|---------|
| 扫描中 | `#6366F1`（默认，未动态绑定） | "● 停止扫描" |
| 未扫描 | `#6366F1` | "● 开始扫描" |

### 3.5 设备列表项交互

```
设备列表项结构:
  Border (圆角6, Padding="12,10")
    ├── Hover: 背景 #00FFFFFF → #1AFFFFFF
    ├── 设备图标: 紫色圆底 (#1A6366F1) + 蓝牙图标 (#6366F1)
    ├── 设备信息: 名称 (14px, 白色, Medium) + ID (10px, #4DFFFFFF)
    └── 信号强度: dBm 文字 + 三角形指示器（透明度由 RSSI 映射）
```

**信号强度透明度映射：**
```
RSSI -30 dBm → Opacity 1.0（极强）
RSSI -60 dBm → Opacity 0.5（中等）
RSSI -90 dBm → Opacity 0.2（极弱）
公式: opacity = clamp((rssi + 90) / 60.0, 0.2, 1.0)
```

## 4. SettingsWindow 交互逻辑

### 4.1 数据流

```
打开窗口:
  MainWindow.OnSettingsClick
    → App.Services.GetService<SettingsWindow>()
    → ViewModel 构造时 LoadFromService() 从 ISettingsService 读取当前值
    → ShowDialog()

修改设置:
  用户操作控件 → 绑定自动更新 ViewModel 属性
  特殊: OverlayOpacity 使用 UpdateSourceTrigger=PropertyChanged 实时预览

保存:
  点击 "保存" → SaveCommand → SaveAsync
    ├── 将 ViewModel 属性写回 ISettingsService
    ├── ISettingsService.SaveAsync() 持久化
    └── 日志记录

取消:
  点击 "取消" / 关闭按钮 → Close()
    └── ViewModel 属性丢弃（未持久化）

恢复默认:
  点击 "恢复默认" → ResetDefaultsCommand
    → ISettingsService.ResetToDefaults() + LoadFromService()
    └── 界面立即刷新为默认值
```

### 4.2 各设置项交互规则

#### 连接设置

| 设置项 | 控件类型 | 绑定属性 | 即时效果 |
|--------|----------|----------|----------|
| 自动连接上次设备 | ToggleSwitch | `AutoConnect` | 仅保存后生效（下次启动时） |

#### 心率警报设置

| 设置项 | 控件类型 | 绑定属性 | 即时效果 |
|--------|----------|----------|----------|
| 通知提醒 | ToggleSwitch | `EnableNotifications` | 仅保存后生效 |
| 声音警报 | ToggleSwitch | `EnableSoundAlert` | 仅保存后生效 |
| 心率上限阈值 | TextBox | `HighHeartRateThreshold` | 仅保存后生效 |
| 心率下限阈值 | TextBox | `LowHeartRateThreshold` | 仅保存后生效 |

#### 外观设置

| 设置项 | 控件类型 | 绑定属性 | 即时效果 |
|--------|----------|----------|----------|
| 主题 | ComboBox | `Theme` | 仅保存后生效（当前未实现浅色主题切换） |
| 窗口不透明度 | Slider | `OverlayOpacity` | **实时预览**（通过 `OnOverlayOpacityChanged` 即时通知） |
| 界面语言 | ComboBox | `Language` | 仅保存后生效（当前未实现动态切换） |
| 极简模式 | ToggleSwitch | `MinimalMode` | 仅保存后生效 |

#### 系统设置

| 设置项 | 控件类型 | 绑定属性 | 即时效果 |
|--------|----------|----------|----------|
| 开机自动启动 | ToggleSwitch | `StartWithWindows` | 仅保存后生效 |
| 最小化到系统托盘 | ToggleSwitch | `MinimizeToTray` | 仅保存后生效 |
| 数据保留天数 | TextBox | `DataRetentionDays` | 仅保存后生效 |

### 4.3 不透明度实时预览机制

```
Slider Value 变化
  → OverlayOpacity 属性 setter
    → OnOverlayOpacityChanged (partial method)
      ├── ISettingsService.OverlayOpacity = value（内存中立即更新）
      └── ISettingsService.NotifySettingsChanged()
        → MainWindow 中 OverlayOpacity 绑定刷新
          → OverlayOpacityToBrushConverter 重新计算
            → MainBorder.Background 更新
```

这是唯一一个具有实时预览效果的设置项。

### 4.4 按钮交互

| 按钮 | 位置 | 样式 | 交互 |
|------|------|------|------|
| 恢复默认 | 底部左侧 | FooterButton, 透明背景, #99FFFFFF 文字, #33FFFFFF 边框 | 立即重置所有值 |
| 取消 | 底部右侧（左） | 同上 | 关闭窗口，不保存 |
| 保存 | 底部右侧（右） | FooterButton, #6366F1 背景, 白色文字, 无边框 | 持久化所有设置 |

### 5.2 Converter 转换器职责

| Converter | 输入类型 | 输出类型 | 触发时机 |

## 7. 异常与边界处理

### 7.1 当前已实现的异常处理

| 场景 | 处理方式 |
|------|----------|
| 启动失败 | `MessageBox` 显示错误 + `Shutdown(1)` |
| 蓝牙权限拒绝 | `MessageBox` 提示 + 跳过 BLE 初始化 |
| 扫描失败 | 日志记录 + 状态回退到 Disconnected |
| 连接失败 | `MessageBox` 显示错误信息 |
| 自动重连失败 | 日志记录（Warning 级别） |
| 窗口位置异常 | 仅在 `WindowLeft > 0 && WindowTop > 0` 时恢复 |
| 数据加载失败 | 日志记录，统计数据保持默认值 |

### 7.2 缺失的异常处理（待优化）

| 场景 | 当前行为 | 建议改进 |
|------|----------|----------|
| 设置保存失败 | 仅日志记录 | 应提示用户 |
| 设备断开连接 | 状态回退，无提示 | 应弹出通知或尝试重连 |
| 阈值输入非法值 | 无验证 | 应限制输入范围（如 30–250） |
| 数据保留天数非法 | 无验证 | 应限制为正整数 |
| 图表数据溢出 | 硬编码 120/60 上限 | 可考虑配置化 |

## 9. 交互流程图

### 9.1 应用启动流程

```
App.OnStartup
  │
  ├─→ DI 注册
  ├─→ 数据库初始化
  ├─→ 设置加载
  ├─→ MainWindow 显示
  │
  ├─→ 蓝牙权限请求
  │     ├─ 成功 → 检查 AutoConnect
  │     │         ├─ 是 → 后台尝试自动重连
  │     │         └─ 否 → 等待用户操作
  │     └─ 失败 → 提示框 → 跳过
  │
  └─→ 加载统计数据（最近 1 小时）
```

### 9.2 设备连接流程

```
用户点击扫描按钮
  │
  ├─→ DevicePickerWindow 打开
  │     │
  │     ├─→ 用户点击 "开始扫描"
  │     │     └─→ BLE 扫描 → 设备列表实时更新
  │     │
  │     ├─→ 用户点击设备项
  │     │     └─→ DialogResult = true, Close()
  │     │
  │     └─→ 用户点击 "关闭"
  │           └─→ DialogResult = false, Close()
  │
  ├─→ (result == true)
  │     └─→ ConnectAsync(device)
  │           ├─ 停止扫描
  │           ├─ BLE 连接
  │           ├─ 保存 LastDeviceId
  │           └─ 失败 → MessageBox 提示
  │
  └─→ (result == false / null)
        └─→ 不做操作
```

### 9.3 设置修改流程

```
用户点击设置按钮
  │
  └─→ SettingsWindow 打开
        │
        ├─→ 用户修改设置项
        │     ├─ 普通控件 → ViewModel 属性更新（内存中）
        │     └─ 不透明度滑块 → 实时预览 + 内存更新
        │
        ├─→ 用户点击 "保存"
        │     └─→ ViewModel → ISettingsService → 持久化
        │
        ├─→ 用户点击 "取消"
        │     └─→ Close()，变更丢弃
        │
        └─→ 用户点击 "恢复默认"
              └─→ 界面立即刷新为默认值
```
